## Implementation Report (2026-07-19)

### Architecture

- NestJS `AIModule` (`@Global`) with provider registry/factory, Prisma repository, pgvector raw SQL, advisory locks, and email-style job worker polling.
- Business services: quota (advisory lock + RESERVED/COMMITTED/RELEASED ledger), safety, usage, prompts, retrieval/RAG, documents, conversations, features, insights, streaming chat.
- Authenticated POST SSE with AbortController propagation on client disconnect; reservation released on cancel/failure.
- Storage integration: `StorageProvider.downloadBuffer()` + `StorageService.downloadAuthorizedAsset()` for PDF/text RAG indexing.
- Frontend shared `AIWorkspace` with SSE stream client, markdown rendering (`react-markdown`, `remark-gfm`, `rehype-katex`).

### Remaining Technical Debt

- Job worker uses synthetic auth context for async generation jobs; prefer persisted job actor metadata.
- RAG currently indexes uploaded media assets; inline course/lesson text ingestion can be expanded.
- Provider moderation hooks are stubbed for OpenAI-compatible providers.
- Frontend feature selector is a text input; replace with curated feature picker per portal.
- Integration tests for SSE stream and pgvector search require running Postgres with pgvector in CI.
