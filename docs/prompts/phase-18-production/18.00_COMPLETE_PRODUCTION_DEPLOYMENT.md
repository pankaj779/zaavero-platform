# Phase 18 — Complete Production Deployment

**Status:** Complete  
**Date:** 2026-07-20  
**Scope:** Final production preparation for the Graphology Platform (no architecture redesign; reuse all existing modules).

---

## Architecture (unchanged)

```
Web (Vercel / Next.js)
  → apiFetch → NestJS API (Render / Docker)
    → Neon PostgreSQL (pooled + direct)
    → Cloudinary | Resend | Razorpay | AI providers | Zoom / Google Meet

Mobile (Expo EAS)
  → same NestJS /api/v1
```

Providers remain **env-switched** — no code changes to enable OpenAI vs Anthropic, Cloudinary, Resend, Zoom, etc.

## Infrastructure

| Concern | Implementation |
|---------|----------------|
| Health | `/api/v1/health` liveness, `/api/v1/ready` DB readiness, `/api/v1/health/deep` provider snapshot |
| Graceful shutdown | `app.enableShutdownHooks()` + Prisma `$disconnect` on module destroy |
| Compression | `compression` middleware |
| Security headers | `helmet` (HSTS + CSP in production) |
| CORS | Allowlist via `CORS_ORIGIN`, credentials enabled |
| Trusted proxies | `TRUST_PROXY` → Express `trust proxy` |
| Body limits | `BODY_LIMIT_BYTES` on JSON/urlencoded |
| Timeouts | `REQUEST_TIMEOUT_MS` on HTTP server |
| Rate limiting | `@nestjs/throttler` global + tighter `auth` throttle on login/register |
| Logging | Structured JSON HTTP access logs (`requestId`, status, durationMs) |
| Connection pooling | Neon pooled `DATABASE_URL` + `DIRECT_URL`; URL param guidance documented |
| Audit retention | `AUDIT_RETENTION_DAYS` env (ops policy) |
| Backups | `scripts/backup-database.sh` / `restore-database.sh` + Neon PITR docs |
| Docker prod | `docker-compose.prod.yml` healthcheck + postgres tuning |
| Render | `render.yaml` blueprint |
| Vercel | `apps/web/vercel.json` |

## Remaining technical debt

1. Web tokens still mirrored in JS-accessible cookies/localStorage — migrate to **HttpOnly Set-Cookie** via BFF for full cookie security posture
2. In-process email/AI workers should eventually split to dedicated worker dynos at high volume
3. OTEL requires installing `@opentelemetry/sdk-node` (+ exporters) in the production image to fully activate (hook is ready)
4. Replace placeholder EAS `projectId` / store submit Apple IDs before store submission
5. Optional Next.js Sentry SDK wiring on the web app (DSN env reserved)
6. After enabling `.npmrc` `virtual-store-dir-max-length`, run a clean `pnpm install` so Windows Hermes native export can use shorter paths
7. Prefer `next/image` for admin/settings media (existing lint warnings)
