# Role-Based Access Control (RBAC)

Version 1.0

---

## Goal

Implement production-ready authorization using the existing Role and Permission models.

Authentication already exists.

This task determines what authenticated users are allowed to do.

---

## Read First

Read all documentation in /docs.

Read previous engineering tasks.

Follow repository abstractions.

Services must never access Prisma directly.

---

## Scope

Implement only:

- JWT authentication guard
- Roles guard
- Permissions guard
- CurrentUser decorator implementation
- @Roles decorator implementation
- @Permissions decorator implementation
- Permission lookup service
- Authorization middleware
- Tests

Do not build Admin pages or business modules.

---

## JWT Guard

Implement JwtAuthGuard.

Requirements:

- Validate access token
- Load authenticated user
- Attach user to request
- Reject invalid or expired JWTs

---

## Roles Guard

Implement @Roles(...roles).

Examples:

@Roles("Admin")

@Roles("Teacher")

Support multiple roles.

---

## Permissions Guard

Implement @Permissions(...permissions).

Examples:

@Permissions("student.view")

@Permissions("course.create")

Load permissions from the database through repository abstractions.

Do not hardcode permission logic.

---

## CurrentUser Decorator

Implement:

@CurrentUser()

Returns the authenticated user.

Support:

@CurrentUser("id")

@CurrentUser("email")

---

## Authorization Rules

- User must be active
- User must not be soft-deleted
- User must belong to an active organization
- Role and permissions evaluated per organization membership
- Return 401 for unauthenticated requests
- Return 403 for authenticated users without required permissions
