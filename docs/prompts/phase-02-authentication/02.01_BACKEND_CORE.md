# Features To Implement

## 1. Configuration Module

Create a centralized configuration module.

Validate all environment variables using Zod.

Required variables include:

- NODE_ENV
- PORT
- DATABASE_URL
- DIRECT_URL
- JWT_SECRET (placeholder only)
- RESEND_API_KEY (placeholder)
- RAZORPAY_KEY_ID (placeholder)
- RAZORPAY_SECRET (placeholder)

Fail fast if required variables are invalid.

---

## 2. API Versioning

Configure URI versioning.

Example:

/api/v1

Future versions should be easy to introduce.

---

## 3. Global Validation

Use ValidationPipe globally.

Requirements:

- whitelist = true
- transform = true
- forbidNonWhitelisted = true

Never trust client input.

---

## 4. Global Exception Filter

Create a global exception filter.

Every error should return a consistent JSON format.

Example:

{
  "success": false,
  "statusCode": 400,
  "message": "...",
  "timestamp": "...",
  "path": "/api/v1/..."
}

Do not expose stack traces in production.

---

## 5. Response Interceptor

Every successful response should follow:

{
  "success": true,
  "message": "...",
  "data": {},
  "timestamp": "..."
}

Implement a reusable global response interceptor.

---

## 6. Logging

Use NestJS Logger.

Every request should log:

- method
- path
- execution time
- status code

Do not log passwords or secrets.

---

## 7. Request ID

Generate a unique request ID for every request.

Return it in the response headers.

Include it in logs.

---

## 8. Security

Enable:

Helmet

Compression

CORS

Configure sensible defaults.

---

## 9. Swagger

Configure OpenAPI.

Expose documentation at:

/api/docs

Include:

- title
- description
- version

Prepare for future authentication documentation.

---

Move them under versioning if appropriate while preserving compatibility if possible.
