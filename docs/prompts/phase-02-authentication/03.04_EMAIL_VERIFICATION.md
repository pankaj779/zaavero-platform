# ENGINEERING TASK 03.04

# Email Verification

Version 1.0

---

## Goal

Implement secure email verification.

This task enables verification of newly registered users.

Do NOT implement password reset.

Do NOT implement refresh tokens.

---

## Read First

Read all documents inside /docs.

Read previous implementation tasks.

Follow repository abstractions.

Do not access Prisma directly from services.

---

## Scope

Implement only:

- Email verification token generation
- Verification endpoint
- Email sending abstraction
- Token expiry
- Resend verification email
- Tests

---

## Email Provider

Do NOT couple the application to a specific provider.

Create:

EmailService Interface

↓

EmailProvider implementation (Resend or sandbox)

Use dependency injection.

Future providers (SES, SendGrid, Mailgun) should be replaceable.

---

## Registration

After successful registration:

Generate a secure verification token.

Store only the hashed token.

Never store raw verification tokens.

Token expires after 24 hours.

Send verification email.

If email sending fails:

Do NOT roll back user creation.

Log the failure.

---

## Endpoint

GET

/api/v1/auth/verify-email?token=...

---

## Verification Flow

Validate token.

Check expiration.

Find associated user.

Mark

emailVerified = true

Delete or invalidate verification token.

Return success.

---

## Resend Endpoint

POST

/api/v1/auth/resend-verification

Request

{
"email":"..."
}

If already verified

Return success.

Do not reveal whether the email exists.

---

## Security

Never expose token hashes.

Never log tokens.

Use cryptographically secure random tokens.

Hash tokens before storage.

---

## Email Template

Prepare a reusable HTML template.

Keep branding generic.

Do not hardcode URLs.

Use FRONTEND_URL from ConfigModule.

---
