# ENGINEERING TASK 03.06

# Forgot Password

Version 1.0

---

## Goal

Implement secure password recovery.

This task enables users to reset forgotten passwords using one-time reset tokens.

Do not implement frontend pages.

---

## Read First

Read all documentation in /docs.

Read previous engineering tasks.

Follow repository abstractions.

Services must never access Prisma directly.

---

## Scope

Implement only:

- Forgot password endpoint
- Reset password endpoint
- Password reset token generation
- Password reset token validation
- Token expiration
- Password update
- Token invalidation
- Tests

---

## Security Requirements

Never reveal whether an email exists.

Always return the same success response from the forgot-password endpoint.

Generate cryptographically secure random reset tokens.

Store only SHA-256 hashes of reset tokens.

Tokens expire after 30 minutes.

Tokens are single use.

After successful password reset:

- Delete/invalidate all outstanding password-reset tokens.
- Revoke every active refresh token for that user.
- Force re-login on every device.

Do not automatically log the user in.

---

## Endpoints

POST /api/v1/auth/forgot-password

Request

{
  "email": "user@example.com"
}

Always return:

{
  "success": true,
  "message": "If an account exists, password reset instructions have been sent."
}

---

POST /api/v1/auth/reset-password

Request

{
  "token": "...",
  "password": "NewStrongPassword123!"
}

Response

{
  "success": true,
  "message": "Password has been reset successfully."
}

---

## Password Rules

Reuse registration password policy.

Hash passwords using Argon2.

Never reuse the raw password anywhere.

---

## Email

Reuse EmailService.

Send password reset email using a reusable template.

Use FRONTEND_URL to generate reset links.

Do not hardcode URLs.

---

## Database

Create a PasswordResetToken table if one does not already exist.

Suggested fields:

- id
- userId
- tokenHash
- expiresAt
- usedAt
- createdAt

Indexes:

- tokenHash
- userId
- expiresAt
