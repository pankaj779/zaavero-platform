## Security Requirements

Never store raw refresh tokens.

Hash refresh tokens using SHA-256 before persistence.

Each refresh token is single-use.

When a refresh succeeds:

1. Verify incoming refresh token.
2. Validate expiration.
3. Ensure it is not revoked.
4. Issue a new access token.
5. Issue a new refresh token.
6. Revoke the previous refresh token.
7. Store the new hashed refresh token.

This is mandatory token rotation.

## Endpoints

POST /api/v1/auth/refresh

Request

{
  "refreshToken": "..."
}

Response

{
  "success": true,
  "message": "Token refreshed successfully.",
  "data": {
    "accessToken": "...",
    "refreshToken": "...",
    "expiresIn": "15m"
  }
}

---

POST /api/v1/auth/logout

Request

{
  "refreshToken": "..."
}

Logout revokes the refresh token.

Always return success even if the token is already invalid.

Do not reveal token validity.
