## Implementation report (2026-07-19)

### Architecture summary
Provider-agnostic `MeetingModule` mirrors Email/Storage: `MeetingProvider` interface + registry (`ZOOM`, `GOOGLE_MEET`, `SANDBOX`). `MeetingService` orchestrates LiveSession provisioning without branching on vendor details. OAuth tokens and host/passcode secrets use AES-256-GCM via `TOKEN_ENCRYPTION_KEY`. Webhooks use insert-or-fetch + atomic claim (`lockedAt`/`lockedBy`).

### Files created
- `apps/api/src/modules/meeting/**` (providers, services, controllers, repository, DTOs, tests)
- Migration already applied earlier: `packages/database/prisma/migrations/20260719112000_video_meeting_foundation/`

### Files modified
- `apps/api/src/app.module.ts` — registers `MeetingModule`
- `apps/api/src/config/env.schema.ts`, `configuration.ts`, `.env.example` — meeting env vars
- `apps/api/src/modules/live-session/**` — provision/sync/start/end/cancel + hostUrl role gating
- `apps/web` live session mapper/types/API + teacher Start/Cancel actions

### OAuth setup
- Admin: `POST /meetings/integrations/oauth/start` with `{ organizationId, provider, redirectPath? }`
- Callbacks: `/meetings/oauth/zoom|google|sandbox/callback`
- Redirect URIs must match Zoom/Google app settings (`API_URL` + path above)

### Webhook setup
- Zoom: `POST /meetings/webhooks/zoom` (CRC `endpoint.url_validation` + `x-zm-signature`)
- Google: `POST /meetings/webhooks/google` (Calendar push headers)
- Sandbox: `POST /meetings/webhooks/sandbox`

### Deployment variables
- `TOKEN_ENCRYPTION_KEY` (required in production)
- `MEETING_SANDBOX_MODE` (default true outside production; forbidden in production)
- `ZOOM_CLIENT_ID`, `ZOOM_CLIENT_SECRET`, `ZOOM_WEBHOOK_SECRET`
- `GOOGLE_MEET_CLIENT_ID` / `GOOGLE_MEET_CLIENT_SECRET` (fallback: `GOOGLE_CLIENT_*`)
