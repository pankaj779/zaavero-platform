# COMPLETION — 2026-07-19

## Architecture summary

- `MediaAsset` is the organization-scoped source of storage metadata and ownership.
- NestJS routes all signed, multipart, replace, delete, metadata, and entity-list operations through `StorageService` and the injected `StorageProvider`.
- Cloudinary is the production provider; the sandbox provider keeps local development and tests deterministic.
- Auth avatars, organization branding, course/lesson/assignment/submission media, certificates, invoice PDFs, and message attachments use the centralized storage flow.
- The web application uses `StorageApi` and shared upload components/hooks; `/auth/me` now hydrates the persisted `profileImage` into cached sessions and teacher/student profile views.

## Files created, modified, and removed

- Created the storage module (controller, service, providers, repository, DTOs, validation, exceptions, tests), storage web client/mappers/tests, shared media/upload components, and the `MediaAsset` migration.
- Modified the database schema, API entity integrations, auth session response, portal upload surfaces, environment examples, and environment documentation.
- Removed stale media/upload placeholder copy and the unused public Cloudinary web variable. No duplicate upload helper required removal; uploads already converge on `StorageApi`.

## Verification

- `pnpm --filter @graphology/api typecheck`, `lint`, and `test`: passed (59 test files passed, 2 skipped; 414 tests passed, 18 skipped).
- `pnpm --filter @graphology/web typecheck`, `lint`, `test`, and `build`: passed (50 test files and 232 tests passed). Lint reports four existing `no-img-element` warnings and no errors.
- `pnpm --filter @graphology/database typecheck` and `lint`: passed.
- Root `pnpm typecheck`, `pnpm lint`, `pnpm test`, and `pnpm build`: passed. Turbo reports only output-configuration warnings for the mobile/tests build tasks.

## Deferred debt

- Add announcement images and calendar event attachments only when those domains gain media fields; no speculative schema or module was introduced.
- A dedicated lesson ZIP upload affordance remains optional; ZIP is accepted by the centralized storage validation and can be wired into the lesson UI when product requirements are defined.
- Production deployment still requires real `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, and `CLOUDINARY_API_SECRET` credentials, with sandbox mode disabled.
