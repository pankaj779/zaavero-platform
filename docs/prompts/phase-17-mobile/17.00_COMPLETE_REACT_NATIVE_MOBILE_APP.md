# Phase 17 — Complete React Native Mobile Application

**Status:** Complete  
**Date:** 2026-07-19  
**Package:** `@graphology/mobile` (`apps/mobile`)

---

## Goal

Build a production React Native (Expo) application for the Graphology Platform that uses the **exact same NestJS backend** as the web app — Auth, RBAC, DTOs, business logic, Cloudinary, Razorpay, Zoom/Meet/Sandbox, AI platform, email, and PDF/certificate systems — with **no duplicated backend logic**, **no new APIs**, and **no mock data**.

Architecture:

```
React Native (Expo Router)
  → Shared mobile API layer (apiFetch)
  → Existing NestJS /api/v1
  → Existing Auth / RBAC / DTOs / business logic
```

## What was built

### 1. Scaffold (`apps/mobile`)

- Expo app with monorepo Metro config (workspace root watchFolders)
- Workspace deps on `@graphology/types`, `@graphology/auth`, `@graphology/utils`
- ESLint via `@graphology/config/eslint/react-library`
- Scripts: `start`, `dev`, `typecheck`, `lint`, `test`, `build`, `native:prebuild`
- Env: `EXPO_PUBLIC_API_URL` (same NestJS base URL as web)

### 2. Shared API layer

| File | Role |
|------|------|
| `lib/api/client.ts` | `apiFetch()` — bearer attach, single 401 refresh, envelope unwrap |
| `lib/api/endpoints.ts` | Courses, lessons, enrollments, assignments, live sessions, attendance, certificates, calendar, notifications, messaging, payments, AI, admin, storage |
| `lib/api/ai-stream.ts` | Authenticated SSE streaming via `expo/fetch` → `/ai/student/chat/stream` |
| `lib/api/errors.ts` | Network / Unauthorized / Forbidden / TokenExpired |
| `lib/security/certificate-pinning.ts` | Pinning-ready `getSecureFetch()` single entry point |

All endpoints mirror the web client's NestJS paths. No forked DTOs or business rules.

### 3. Auth

| Capability | Implementation |
|------------|----------------|
| Login / Register | `authApi` + `authService` → `/auth/login`, `/auth/register` |
| Forgot / Reset password | `/auth/forgot-password`, `/auth/reset-password` |
| Email verification | `/auth/verify-email`, `/auth/resend-verification` |
| Refresh tokens | Single-flight refresh (same pattern as web) |
| Logout | Clears SecureStore + server logout |
| Secure token storage | Expo Secure Store + in-memory cache |
| Biometric unlock | `expo-local-authentication` + preference flag |
| Session restore | Hydrate SecureStore → refresh → `/auth/me` |

### 4. Role-based navigation

| Portal | Route group | Guard |
|--------|-------------|-------|
| Auth | `/(auth)` | Redirects away when authenticated |
| Student | `/(student)` | `RoleGuard role="Student"` |
| Teacher | `/(teacher)` | `RoleGuard role="Teacher"` |
| Admin | `/(admin)` | `RoleGuard role="Admin"` |

Entry (`app/index.tsx`) restores the session and routes by role priority: Admin → Teacher → Student.

### 5. Student app screens

Dashboard · Courses · Course Details · Lesson Player · Assignments · Live Classes · Attendance · Progress · Certificates · Calendar · Notifications · Messages · Profile · Settings · Payments · AI Tutor · Downloads

### 6. Teacher app screens

Dashboard · Courses · Lessons · Assignments · Students · Attendance · Live Classes · Certificates · Calendar · Messages · Notifications · Analytics · Profile · Settings · AI Workspace

### 7. Admin app screens

Dashboard · Users · Teachers · Students · Organization · Analytics · Audit Logs · Payments · Notifications · Settings · Academic · AI Admin

### 8. Cross-cutting features

| Feature | Reuse path |
|---------|------------|
| Media / avatars | `POST /storage/uploads` + `PATCH /auth/me/avatar` (Cloudinary-backed) |
| Payments | `GET /payments/config`, `POST /payments/orders`, `POST /payments/verify`, invoices |
| Video meetings | Live-session start/end/join → Zoom / Google Meet / Sandbox URLs |
| AI | Phase 16 streaming tutor + teacher/admin endpoints; role restrictions enforced by backend |
| Email | Existing auth verification / forgot-password email flows |
| PDF | Certificate + invoice download via authenticated FileSystem downloads |
| Push | Expo Notifications → `POST /notifications/devices` |
| Offline | React Query persistence + paused mutation resume |
| Downloads | Offline lessons / PDFs / certificates / invoices (local manifest) |
| Camera | Avatar capture + document/assignment image pick |
| Performance | FlashList, Expo Image memory-disk cache, React Query prefetch/staleTime |
| Security | SecureStore only for secrets; certificate-pinning ready; client role guards + server RBAC |

## Explicit non-goals / constraints honored

- **No new NestJS APIs** — mobile only consumes existing `/api/v1` routes.
- **No mock data / fake providers** — Cloudinary, Razorpay, Zoom/Meet/Sandbox, AI providers remain backend-owned.
- **No forked business logic** — quotas, RBAC, certificate issuance, payments verification stay on the server.
- **No commit / no push** — as requested.

## Notes

1. **Hermes on Windows:** Production export uses `"jsEngine": "jsc"` because the Windows `hermesc.exe` binary was not present in this environment. EAS / macOS CI builds can switch back to Hermes (`"jsEngine": "hermes"`) when the compiler is available.
2. **Device API URL:** Physical devices must set `EXPO_PUBLIC_API_URL` to the machine LAN IP (not `localhost`).
3. **Native Razorpay sheet:** Checkout creates the order through the shared backend; a native `react-native-razorpay` sheet can wrap `config.keyId` + `order.providerOrderId` without changing the API contract.
4. **Push project ID:** Production Expo push requires an EAS `projectId` in `app.json` → `extra.eas.projectId`.
