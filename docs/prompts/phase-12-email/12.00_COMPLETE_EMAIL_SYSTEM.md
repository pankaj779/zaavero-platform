# COMPLETION — 2026-07-19

## Shipped

- Tenant-scoped Prisma foundation for logs, versioned templates, durable queue/outbox rows, preferences, attachments, provider events, and invitations, with 25 seeded system templates.
- Central `EmailService` queue, worker retry/backoff/dead-letter flow, preference enforcement, Resend and local sandbox providers, signed webhook processing, replay/idempotency protection, and admin observability.
- `BusinessEmailService` integrations for authentication, payments, subscriptions, refunds, certificates, notifications, enrollments, assignments/submissions, and live sessions.
- Organization, teacher, and student invitation creation, authenticated recent-invitation listing, resend/revoke, and public token acceptance.
- API-backed student and teacher email preferences/verification UI, teacher student-invitation UI, admin email workspace/invitations, and invitation acceptance page.
- Cleanup removed the obsolete provider service reference and unused pre-queue verification/password-reset template helpers. Environment references now consistently document `RESEND_WEBHOOK_SECRET` and all `EMAIL_*` settings.

## API Surface

- `GET|PATCH /api/v1/email/preferences`
- `GET /api/v1/email/admin/logs|queue|failed|templates|provider|stats`
- `POST /api/v1/email/admin/queue/:id/cancel|retry`
- `POST /api/v1/email/admin/templates/preview`
- `POST /api/v1/email/webhooks/resend`
- `GET|POST /api/v1/email/invitations`
- `POST /api/v1/email/invitations/:id/resend|revoke`
- `POST /api/v1/email/invitations/accept`
- Existing auth endpoints enqueue centralized `welcome`, `verify_email`, `forgot_password`, and `password_changed` templates.

## Events → Templates

- Registration/authentication → `welcome`, `verify_email`, `forgot_password`, `password_changed`
- Invitation create/resend → `teacher_invitation`, `student_invitation`, `organization_invitation`
- Payment capture/invoice/subscription lifecycle/refund → `payment_successful`, `invoice_available`, `subscription_started`, `subscription_renewed`, `subscription_cancelled`, `refund_processed`
- Certificate issue → `certificate_issued`
- Email-channel notification → `announcement` or `general_notification`
- Enrollment → `course_enrollment`, `batch_enrollment`
- Assignment publish/submit/grade → `assignment_published`, `assignment_submitted`, `assignment_graded`
- Live-session create/cancel → `live_reminder`, `live_cancelled`

## Portal Routes

- Student: `/dashboard/settings`
- Teacher: `/teacher/settings`
- Admin: `/admin/email` (linked from `/admin/settings`)
- Public invitation acceptance: `/accept-invitation`
